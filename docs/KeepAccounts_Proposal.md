# KeepAccounts: AI 智能对话记账 App 项目企划书

> 文档状态（2026-03）：本文保留了早期产品构想。当前实际执行路线已切换为 Android 原生 Local-First。
> 以 `docs/prd.md`、`docs/TRD.md`、`docs/API_Docs.md` 与开发路线文档为准。

## 1. 产品愿景 (Product Vision)
传统的记账软件需要繁琐的表单输入（选分类、填金额、填日期、写备注），导致用户难以坚持。本项目旨在打造一款**像微信聊天一样自然**的记账App。用户只需通过自然语言描述消费行为，AI管家会自动完成意图识别、金额提取、分类打标，并以拟人化的口吻进行反馈。同时提供专业的财务可视化报表，彻底消除记账的阻力。

---

## 2. 核心应用场景与功能 (Core Features)

### 2.1 核心：智能对话记账 (AI Chat Input)
*   **自然语言输入**：当前以文字输入为主，语音能力优先复用系统输入法，不单独提供应用内 Mic。
*   **自动解析与分类**：AI后台自动提取时间（今天中午）、金额（35）、分类（餐饮/快餐）、备注（肯德基）。
*   **拟人化反馈**：系统以人类口吻回复：“已为您记录：餐饮支出 35 元。您本周餐饮预算还剩 150 元，请注意保持哦~”
*   **多笔连记/含糊纠正**：AI支持多轮对话修改（“不对，那笔是30元”）。

### 2.2 视图与统计 (Views & Analytics)
*   **日历视图 (Calendar View)**：日历上直观显示每天的“总收入”与“总支出”，点击具体日期可查看当天的消费明细流。
*   **预算与超支预警 (Budget Management)**：提供按周、按月的预算设置，进度条展示当前结余，超额时AI主动提醒。
*   **专业统计报表 (Statistical Reports)**：
    *   **分类饼图**：吃喝、交通、购物等各项支出的占比。
    *   **趋势折线图**：展示近几个月/几周的收支起伏趋势。

### 2.3 AI 人性化与直觉交互设计要点
*   **最小打扰澄清**：仅在关键信息缺失时提问，优先给出候选确认（如“按餐饮 35 元记账可以吗？”）。
*   **回执即编辑**：记账成功卡片内直接提供修改/撤销，不强迫用户跳转到二级页面。
*   **错误低成本修复**：支持“改成 30”“分类改交通”等自然语言二次修正。
*   **建议可解释可关闭**：预算提醒与消费建议必须可解释来源，且允许用户关闭或降频。
*   **一致语气与节奏**：AI 回复要简洁、温和、可预期，避免过度打扰和冗长说教。

## 3. 当前技术路线 (Current Stack)

### 3.1 客户端与状态管理
*   **平台**：Android 原生（Kotlin + Jetpack Compose）。
*   **导航与页面结构**：Navigation Compose + 多 Tab 页面。
*   **状态管理**：ViewModel + Coroutines + StateFlow（UDF）。

### 3.2 数据与存储
*   **主数据源**：Room（交易流水、聊天消息）。
*   **用户偏好**：DataStore（主题、预算、提醒、AI配置、分类）。
*   **架构原则**：Local-First，后续在用户授权登录后提供可选云同步。

### 3.3 AI 能力层
*   **模型接入**：SiliconFlow 流式对话接口。
*   **通信方式**：Retrofit `@Streaming` + 增量解析 `data:` 事件。
*   **结构化回执契约**：通过 `<DATA>...</DATA>` 解析 `amount/category/desc/recordTime/date`。

## 4. 开发阶段更新 (Development Phases)

### Phase 1（已完成）：核心账本闭环
*   Room + DataStore 落地。
*   手动记账、首页/账本/搜索主链路打通。

### Phase 2（已完成）：AI 对话记账闭环
*   流式对话接入。
*   `<DATA>` 回执解析与交易落库。
*   聊天回执与账本编辑跳转联动。

### Phase 3（进行中）：时间一致性与账本稳定性
*   `recordTime` 全链路对齐。
*   账本月/年过滤下沉到 ViewModel。
*   当前月趋势图未来日期留白。

### Phase 4（待启动）：能力补齐
*   核心操作极简化 + AI 人性化交互 V1。
*   账号登录与身份绑定预研（含微信绑定）。
*   可选云同步 PoC（冲突合并与离线回放策略）。
*   图片/OCR 记账能力（价值验证后推进）。
*   导入导出作为备选迁移方案（低优先级）。
*   Android 16 Rich Ongoing Notifications（低优先级）。

---

## 5. 接下来优先事项

1. **核心操作极简化 + 分类增强**：降低认知成本，优先保证“傻瓜式”一键完成核心路径。
2. **AI 人性化交互 V1**：补齐澄清、确认、纠错与撤销，提升可控性与直觉感。
3. **账号与微信绑定预研**：先产出技术选型与合规方案，再进入开发。
4. **可选云同步 PoC**：在 Local-First 前提下验证多设备一致性与冲突策略。
5. **低优先级能力池**：图片/OCR、导入导出、Rich Notifications 按价值与资源择优推进。
