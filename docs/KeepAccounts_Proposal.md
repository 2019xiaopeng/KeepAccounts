# KeepAccounts: AI 智能对话记账 App 项目企划书

## 1. 产品愿景 (Product Vision)
传统的记账软件需要繁琐的表单输入（选分类、填金额、填日期、写备注），导致用户难以坚持。本项目旨在打造一款**像微信聊天一样自然**的记账App。用户只需通过自然语言描述消费行为，AI管家会自动完成意图识别、金额提取、分类打标，并以拟人化的口吻进行反馈。同时提供专业的财务可视化报表，彻底消除记账的阻力。

---

## 2. 核心应用场景与功能 (Core Features)

### 2.1 核心：智能对话记账 (AI Chat Input)
*   **自然语言输入**：支持文字和语音输入。例如：“中午吃了肯德基，花了35块钱”、“昨天买衣服超支了，花了500”。
*   **自动解析与分类**：AI后台自动提取时间（今天中午）、金额（35）、分类（餐饮/快餐）、备注（肯德基）。
*   **拟人化反馈**：系统以人类口吻回复：“已为您记录：餐饮支出 35 元。您本周餐饮预算还剩 150 元，请注意保持哦~”
*   **多笔连记/含糊纠正**：AI支持多轮对话修改（“不对，那笔是30元”）。

### 2.2 视图与统计 (Views & Analytics)
*   **日历视图 (Calendar View)**：日历上直观显示每天的“总收入”与“总支出”，点击具体日期可查看当天的消费明细流。
*   **预算与超支预警 (Budget Management)**：提供按周、按月的预算设置，进度条展示当前结余，超额时AI主动提醒。
*   **专业统计报表 (Statistical Reports)**：
    *   **分类饼图**：吃喝、交通、购物等各项支出的占比。
    *   **趋势折线图**：展示近几个月/几周的收支起伏趋势。

## 3. 技术栈推荐 (Tech Stack Recommendation)

为了实现手机App端、顺畅的UI交互以及强大的AI处理能力，推荐以下现代化的技术栈组合：

### 3.1 移动端 (Frontend / Mobile App)
*   **框架**：**Flutter** 或 **React Native**（跨平台开发，一套代码编译 iOS 和 Android）。推荐 Flutter，它的UI渲染性能和动画表现极佳，适合做精美的图表和日历。
*   **图表库**：`fl_chart` (Flutter) 或 `react-native-chart-kit`，用于绘制饼图和折线图。
*   **状态管理**：Riverpod (Flutter) 或 Redux/Zustand (React)。

### 3.2 后端服务 (Backend Service)
*   **框架**：**Python + FastAPI**。因为本项目深度依赖AI和LLM，Python生态最成熟；FastAPI性能极高且支持异步，特别适合处理流式(Streaming)的对话响应。
*   **数据库**：**PostgreSQL**。记账涉及严格的金额计算和时间维度查询，关系型数据库最稳妥。
*   **ORM框架**：SQLAlchemy 或 SQLModel。

### 3.3 人工智能层 (AI / LLM Layer)
*   **大语言模型**：接入 OpenAI (GPT-4o-mini) / DeepSeek / 阿里通义千问 等 API。这类模型处理文本分类和JSON数据提取既快又便宜。
*   **逻辑控制**：使用 **LangChain** 封装 Prompt，规定 AI 的输出要求。除了提供给用户的多段感情回复，它必须返回结构化的账单 JSON，用于后台存储。
*   **账单数据结构 (JSON Schema)**：
    AI 每次成功解析自然语言后，都应生成如下标准的 JSON 格式：
    ```json
    {
      "transaction_type": "expense",      // 取值: expense (支出), income (收入)
      "amount": 15.00,                    // 消费金额，处理为标准浮点数
      "category": "餐饮",                 // 对应的一级分类
      "sub_category": "零食/甜点",        // 对应的二级分类
      "remark": "吃了冰淇淋",             // 用户原始表达或摘要
      "transaction_date": "2026-03-26",   // 实际消费发生的日期(YY-MM-DD)，处理“昨天/刚买的”等语义
      "record_time": "2026-03-26T14:30:00Z" // 系统真实生成的记录时间戳
    }
    ```

## 4. 开发流程设计 (Development Phases)

项目可以采用敏捷开发，小步快跑，分为以下四个阶段：

### Phase 1: MVP（最小可行性产品）- 核心记账
*   **目标**：跑通“对话 -> AI解析 -> 存入数据库 -> 返回确认”的全链路。
*   **任务**：搭建基础数据库（用户表、账单表）；开发大模型 Prompt 提取逻辑；开发类似微信聊天的简单UI界面。

### Phase 2: 数据展示与报表 - 统计分析
*   **目标**：让用户看到钱花在了哪里。
*   **任务**：在App中引入日历组件，渲染每日收支；引入图表库，实现按周/月的饼图和分类统计排行。

### Phase 3: 预算管理与用户系统
*   **目标**：补齐记账软件的进阶功能。
*   **任务**：支持用户注册登录保存数据到云端；设置月度/周度预算；在AI的 Prompt 中加入当前预算上下文，让AI计算并给出超支提醒。

### Phase 4: 体验进阶升级
*   **目标**：引入多模态，打造终极懒人体验。
*   **任务**：接入语音转文字 (ASR)，支持直接按住说话记账；接入 OCR 图像识别，支持拍购物小票自动记账。

## 5. 项目启动的第一步
如果你对以上的设想满意，我们可以按照下面的顺序开工：
1. **数据库设计**：先设计核心的 `账单 (Transaction)` 和 `分类 (Category)` 数据表结构。
2. **AI Prompt 调试**：先写一段 Python 脚本，测试怎么让大模型稳定地把自然语言转化为结构化的记账数据。
3. **搭建后端 API**：用 FastAPI 把上述逻辑封装成接口。
4. **App 界面开发**：画聊天界面和图表。
