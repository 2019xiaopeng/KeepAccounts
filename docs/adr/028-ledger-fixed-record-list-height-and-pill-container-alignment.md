# ADR 028: 账本明细固定位高与药丸容器贴边对齐

- 日期: 2026-03-29
- 状态: Accepted

## 背景

在 v1.1.0 发布前细节回归中，账本页仍有两处体验不一致：

1. 账本明细在分页后会随当前记录条数变化而改变模块高度，造成页面抖动感。
2. 药丸切换组件中，内层选中胶囊与外层容器在垂直方向存在不对称贴边（上方贴边、下方留缝）。

## 决策

1. 账本明细区域采用固定“5 条记录位高”布局：
   - 保持每页最多 5 条数据。
   - 容器高度按 5 条行高 + 间距 + 内边距固定计算。
   - 当本页记录不足 5 条时补齐占位行，避免整体高度变化。
2. 药丸组件的选中胶囊不再仅依赖容器总高度，而是基于被选中子项的真实 `y` 与 `height` 定位，确保上下贴边一致。

## 影响

- 账本页滚动和翻页视觉更稳定，不因数据条数波动而跳动。
- 药丸组件在不同文本大小与配置下垂直对齐一致，细节观感更统一。

## 代价与权衡

- 引入占位行会在少数据页保留空白区域，但换来稳定布局和更明确的分页预期。
- 药丸定位逻辑增加少量测量状态（top/height），但复杂度可控，收益明显。
