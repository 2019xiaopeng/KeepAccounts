# ADR 028: 账本明细固定五条位高与药丸切换器上下对称对齐

- 日期: 2026-03-29
- 状态: Accepted

## 背景

在发布前回归中，账本页仍存在三类细节问题：

1. 账本明细区域会随当前页记录数量变化高度，导致页面观感跳动。
2. 顶部药丸切换器中，选中胶囊与外层容器上下贴边不对称（顶部更贴近，底部留缝）。

3. 分页按钮在可翻页与不可翻页状态下缺少清晰视觉反馈。

同时，在明细列表中观察到末条文本存在被视觉遮蔽/裁切风险，主要与列表容器高度策略和行内容空间不足有关。

## 决策

1. 账本明细采用严格固定高度匹配法：
   - 单条记录强制固定高度为 68dp。
   - 列表容器强制固定为 `68dp * 5`，不再依赖内容自撑开。
   - 记录不足 5 条时补空槽位，保证模块高度恒定。
2. 单条记录内部使用垂直居中布局（`Column(verticalArrangement = Arrangement.Center)`），并约束次行文本单行省略，避免底部裁切。
3. 分页控制区与列表严格容器隔离：分页 Row 固定放在列表下方，禁止覆盖/悬浮在列表上层。
4. 分页按钮新增动态高亮状态：
   - 可翻页时按钮使用主题色背景。
   - 不可翻页时保持浅灰背景与灰色文字。
   - 点击行为通过 `enabled` 状态防护。
5. 药丸切换器改为固定容器高度（36dp）+ 固定高亮高度（32dp）+ 强制垂直居中对齐，消除上下不对称间隙。

## 影响

- 账本页翻页时版面不再跳动，阅读节奏一致。
- 明细末条文本显示更完整，裁切风险显著降低。
- 翻页按钮状态可感知性提升，误触与无效点击成本降低。
- 药丸切换器在不同字号/文案长度下保持稳定的垂直居中观感。

## 代价与权衡

- 固定 5 条位高会在少数据页保留空白区域，但换来可预期且稳定的布局。
- 药丸组件为保证严格对齐，改用固定高与居中规则，减少了自由度但提升了一致性。
