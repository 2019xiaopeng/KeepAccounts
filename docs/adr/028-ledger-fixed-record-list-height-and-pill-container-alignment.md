# ADR 028: 账本明细固定五条位高与药丸切换器上下对称对齐

- 日期: 2026-03-29
- 状态: Accepted

## 背景

在发布前回归中，账本页仍存在两类细节问题：

1. 账本明细区域会随当前页记录数量变化高度，导致页面观感跳动。
2. 顶部药丸切换器中，选中胶囊与外层容器上下贴边不对称（顶部更贴近，底部留缝）。

同时，在明细列表中观察到末条文本存在被视觉遮蔽/裁切风险，主要与列表容器高度策略和行内容空间不足有关。

## 决策

1. 账本明细容器固定为“五条记录位高”：
   - 每页最多显示 5 条。
   - 列表容器按 5 条行最小高度 + 行间距 + 内边距计算固定高度。
   - 当记录不足 5 条时补空槽位，保证页面整体高度稳定。
2. 每条记录使用最小高度而非硬编码死高度，并限制次行文本为单行省略，避免底部文本被裁切。
3. 药丸切换器选中背景块改为使用“选中项真实 y/height”定位，而非仅按容器总高度铺满，修复上下不对称间隙。
4. 顶部切换器外层 Row 显式设置 verticalAlignment = Alignment.CenterVertically，防止容器级对齐偏差。

## 影响

- 账本页翻页时版面不再跳动，阅读节奏一致。
- 明细末条文本显示更完整，裁切风险显著降低。
- 药丸切换器在不同字号/文案长度下保持稳定的垂直居中观感。

## 代价与权衡

- 固定 5 条位高会在少数据页保留空白区域，但换来可预期且稳定的布局。
- 药丸组件增加少量测量状态（top/height），实现复杂度小幅上升，但收益明显。
