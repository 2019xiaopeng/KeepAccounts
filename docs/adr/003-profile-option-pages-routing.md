# 003 - Profile Option Pages Routing

- Status: Accepted
- Date: 2026-03-26

## Context
`.temp` 原型中的 Profile 页面只展示了“分类管理/账本设置/导入导出/清理缓存/主题外观/帮助反馈”入口，没有二级页面。当前 Kotlin 项目需要把这些入口变成可进入的真实界面，同时保持主导航体验不被干扰。

## Decision
1. 使用 Navigation Compose 增加独立二级路由页面，每个选项对应一个 route。
2. 进入二级页面时自动隐藏底部导航，仅保留主四页面显示底栏。
3. 二级页面首版使用静态数据与统一视觉模板，后续再接真实业务逻辑。

## Why
1. 每个选项独立路由有利于后续逐页接入 ViewModel 与业务状态。
2. 隐藏底栏可减少层级混淆，符合“详情页”导航习惯。
3. 先静态实现可以确保 UI 完整交付，不阻塞后端/数据层迭代。

## Consequences
- Profile 入口现在都可点击并进入对应页面。
- 后续接入真实数据时无需更改导航结构，只需替换页面内部内容。
