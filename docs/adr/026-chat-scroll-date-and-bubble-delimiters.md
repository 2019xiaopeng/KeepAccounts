# ADR 026: Chat 自动滚动、回执日期一致性与流式分气泡策略

- 日期: 2026-03-29
- 状态: Accepted

## 背景

发布前回归中发现 3 个体验/一致性问题：

1. 用户发送消息后，聊天列表不会自动回到底部，需手动上滑才能看到新消息。
2. 相对日期（如“昨天”）在交易库中入账正确，但聊天回执卡片与“修改”页日期仍显示今天。
3. AI 流式回复按单换行或细粒度分隔被过度切分，导致语气泡断句生硬。

## 决策

1. 聊天页在消息列表长度/首条变化以及发送中状态变化时，自动滚动到列表底部（reverseLayout 场景下索引 0）。
2. 聊天回执日期不再使用消息时间兜底，优先使用与 transactionId 关联的交易 recordTimestamp；若缺失，再回退 payload date 和消息时间。
3. 手动记账预填模型新增可选 recordTimestamp，编辑页初始化时优先使用该时间戳，保证修改页日期与原记录一致。
4. AI 多语气泡切分改为“显式分隔优先”：以双换行（\n\n）或约定分隔符切分；移除按单换行/逗号切分，避免不自然断句。
5. 系统提示词补充约束：多条独立消息请使用双换行分隔。

## 影响

- 发送后可见性提升，减少“消息是否发送成功”的不确定感。
- 回执卡片、编辑页与账本首页的日期语义一致。
- 流式对话更接近真人连发体验，断句更自然。

## 代价与权衡

- 观察聊天记录时引入交易流 join，增加少量映射开销；以一致性收益换取可接受的成本。
- 显式分隔优先后，AI 若未给出双换行，将倾向单气泡输出；通过提示词引导与后续策略可持续优化。
