# ADR 013: 聊天顶栏一体化、输入区简化与启动页图片策略

- Status: Accepted
- Date: 2026-03-28

## Context
本轮迭代集中在三类体验问题：
1. 聊天页顶部栏希望更接近微信式一体化视觉，状态栏与标题区不应割裂。
2. 聊天输入区当前包含话筒和图片图标，用户希望先简化为纯文本发送，提升主路径效率。
3. 首页 slogan 目前写死，用户希望在设置页可自定义并持久化。
4. 用户提供了启动图片，希望应用具备可落地的启动页图片方案。

## Decision
1. 聊天页顶部采用一体化容器：状态栏内边距 + 顶栏内容 + 底部分割线统一在同一背景层中渲染。
2. 输入区移除话筒和图片图标，只保留文本输入与发送按钮，减少干扰控件。
3. 新增 `home_slogan` 本地持久化字段（DataStore），并在“个人设置”中提供编辑入口。
4. 引入 `SplashActivity` 作为 Launcher 入口，启动图使用 `drawable/splash_launch.png`，支持直接替换资源完成品牌化。

## 操作过程
1. 更新聊天页：
   - 重构顶栏为一体式背景容器。
   - 输入框改为“文本框 + 发送按钮”布局。
2. 更新设置与首页：
   - DataStore 新增 `home_slogan` 读取与保存接口。
   - 首页顶部副标题改为读取设置值。
   - 个人设置页新增 slogan 编辑项并纳入保存流程。
3. 更新启动流程：
   - 新增 `SplashActivity`。
   - `AndroidManifest.xml` 将 Launcher 入口改为 Splash。
   - 新增启动页主题 `Theme.KeepAccounts.Splash` 与窗口背景资源，降低冷启动闪白。

## 判断与取舍
### A. 为什么不在聊天输入区保留更多入口
- 现阶段主诉求是“快速记账对话”，高频路径是文本输入。
- 先收敛交互可降低误触与视觉噪声，后续再按需求回补多媒体入口。

### B. 为什么 slogan 放在个人设置而不是主题设置
- slogan 属于用户表达，不属于视觉主题参数。
- 放在个人设置更符合用户心智，且和昵称/头像归于同类资料。

### C. 为什么采用独立 SplashActivity 而不是仅系统默认启动主题
- 独立 Activity 能稳定承载全屏图片并控制停留时长。
- 资源替换路径清晰：直接替换 `splash_launch.png` 即可。

## Consequences
- 正向：
  - 聊天顶部结构更统一，视觉更稳定。
  - 输入区更聚焦，记账发送路径更短。
  - 首页文案可配置，个性化能力提升。
  - 启动页图片已具备直接替换与发布能力。
- 代价：
  - 启动链路新增一个 Activity，需维护一次页面跳转。
  - 输入区暂不支持语音与图片发送入口。

## Next
1. 根据真实机型反馈微调聊天顶栏高度与透明度。
2. 若后续恢复多媒体入口，建议改为“展开面板”而非常驻图标。
3. 启动图若需适配横屏/平板，可追加不同尺寸资源与裁切策略。
