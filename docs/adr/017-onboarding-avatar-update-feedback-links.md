# ADR 017: 初始化头像持久化、更新检测收敛与反馈入口外链策略

- Status: Accepted
- Date: 2026-03-28

## Context
在准备首版发布前发现以下风险点：
1. 首次初始化页头像直接保存 content URI，重启后可能失效。
2. 更新检测每次启动都请求并可能重复提示同一版本。
3. 帮助与反馈中心仍是静态占位，缺少可执行入口。
4. 个人页记账天数为硬编码，影响数据可信度。

## Decision
1. 初始化头像上传统一改为本地持久化文件存储（与设置页策略一致）。
2. 更新检测策略加入两层收敛：
   - 最小检查间隔（12 小时）。
   - 同版本提示去重（提示后不重复弹同一 tag）。
3. 帮助与反馈中心统一外链到 GitHub Issues（列表、新建问题、功能建议）。
4. 个人页“已坚持记账 X 天”改为基于真实交易日期集合计算。

## Consequences
- 正向：
  - 降低初始化头像失效概率。
  - 减少启动时不必要网络请求与重复打扰。
  - 反馈入口可直接闭环到仓库 issue 流程。
  - 个人页数据展示更可信。
- 代价：
  - 更新检测增加少量本地偏好状态维护。

## Next
1. 后续可新增“忽略该版本”与“手动检查更新”入口。
2. 可在反馈入口增加 issue 模板参数，进一步规范提交信息。
