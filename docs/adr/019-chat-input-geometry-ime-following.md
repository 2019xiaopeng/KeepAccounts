# ADR 019: 聊天输入栏几何贴合与 IME 跟随策略

- Status: Accepted
- Date: 2026-03-29

## Context
在聊天页细节回归中，出现三类体验偏差：
1. 输入框胶囊与发送按钮圆形弧度不一致，视觉上存在“上下厚、右侧薄”的割裂。
2. 输入栏与底部主导航之间留白偏大，整体看起来悬空。
3. 唤出键盘后输入栏未随 IME 抬升，导致输入内容可能被遮挡。

目标是让交互接近微信/QQ 这类 IM 产品：贴底、顺滑、键盘优先可见。

## Decision
1. 输入栏几何统一：
   - 发送按钮调整为 `44dp` 圆形。
   - 文本输入容器最小高度调整为 `44dp`，圆角半径 `22dp`，与发送按钮同半径体系。
   - 外层输入栏容器圆角收敛为 `24dp`，并降低内边距，减少厚重感。
2. 底部位置下移：
   - 移除输入栏的 `navigationBarsPadding`，避免在主导航场景下重复抬升。
   - 将外部纵向边距从较大值收敛为 `4dp` 级别，让输入栏更贴近底部导航区域。
3. 键盘跟随策略：
   - 在输入栏容器启用 `imePadding`，确保键盘弹出时输入栏随 IME 上移。
   - 保持页面主结构不做额外浮层叠加，减少重复 inset 导致的位置漂移。

## Why Not
- 为什么不同时启用 `navigationBarsPadding + imePadding`：
  - 在当前主导航已占位的布局中会产生重复底部补偿，表现为输入栏离底部过远。
- 为什么不继续使用高输入框高度：
  - 会放大与发送按钮的几何不一致，破坏聊天输入区“单行优先、自然扩展”的观感。

## Consequences
- 正向：
  - 输入框与发送按钮弧度统一，视觉更协调。
  - 非键盘态更贴底，减少“悬空”感。
  - 键盘态可稳定看到输入区域，符合聊天主路径。
- 代价：
  - 对不同机型系统导航模式仍需回归（手势导航/三键导航）。

## Validation
- 本轮已通过：
  - `:app:compileDebugKotlin`
  - `:app:assembleDebug`

## Next
1. 若后续希望达到更接近 IM 产品的体验，可在键盘弹出时临时隐藏底部主导航。
2. 可增加输入栏高度动画（软键盘弹出/收起）以进一步提升跟手感。
