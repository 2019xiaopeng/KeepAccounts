# ADR 018: 全局点击反馈去 Ripple、聊天输入栏重构与键盘顶起修复

- Status: Accepted
- Date: 2026-03-29

## Context
上一轮交互中，围绕聊天与整体触感提出了三项明确诉求：
1. 全 App 去除点击 Ripple 灰色波纹，改为「按下缩小、松开回弹」的轻反馈。
2. 聊天页输入区改为更贴近 IM 场景的胶囊样式（更弱边界、更强沉浸感）。
3. 点击输入框时，键盘不应把页面整体顶上去导致顶部用户/标题区域不可见。

同时在实施批量替换时暴露过一次中文编码污染风险，需要在方案层面约束后续脚本策略。

## Decision
1. 新增统一交互修饰器 `appPressable`，并在 UI 层批量替换 `clickable`：
   - 默认禁用视觉 indication（即无 Ripple）。
   - 基于按压态执行轻微缩放动画（默认 `pressedScale=0.975`）。
2. 聊天输入栏重构为毛玻璃胶囊容器：
   - 左侧图片入口（预留多模态入口位置）。
   - 中部多行输入（1~4 行）。
   - 右侧圆形发送按钮。
   - 占位文案改为「发送消息给 [AI 名称+头像]...」。
3. 键盘行为采用双层保障：
   - `AndroidManifest` 中 `MainActivity` 设置 `windowSoftInputMode="adjustResize"`。
   - 输入栏增加 `navigationBarsPadding + imePadding`，确保仅底部区域抬升。
4. 编码安全策略：
   - 后续批量替换统一使用 UTF-8 显式读写，避免 PowerShell 默认代码页导致中文乱码。

## Why Not
- 为什么不继续保留 Ripple：
  - 在当前浅色玻璃态视觉中，Ripple 灰闪烁感较突兀，破坏沉浸性。
- 为什么不只改聊天页而不做全局交互统一：
  - 触感不一致会产生「页面割裂」，用户感知成本更高。
- 为什么不只靠 `adjustPan`：
  - `adjustPan` 容易导致顶部信息被挤出视野，不符合聊天场景稳定预期。

## Consequences
- 正向：
  - 全局触感一致，点击反馈更柔和。
  - 聊天输入区视觉与交互更接近主流 IM 心智。
  - 键盘弹出时顶部内容保持可见，输入路径更稳定。
- 代价：
  - 批量替换涉及多个 UI 文件，需在每轮后做编译回归。
  - 新增公共修饰器后，后续交互需要遵循统一入口约束。

## Next
1. 若要启用图片发送能力，可直接在输入栏左侧入口接入现有 URI 流。
2. 可为 `appPressable` 增加统一按压时长与可配置动画曲线，便于主题化。
3. 在 CI 增加一次 `:app:assembleDebug` 检查，防止交互批量改动后回归遗漏。
