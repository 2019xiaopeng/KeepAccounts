# ADR 016: 聊天发送按钮修复、账本分段切换无动画化与首次初始化优化

- Status: Accepted
- Date: 2026-03-28

## Context
收尾阶段出现 4 个体验问题：
1. 聊天发送按钮中的“发送”文案出现上下换行。
2. 账本页多个分段切换（日历/统计、月度/年度、支出/收入/结余、排序）动画过多，滚动回流时有视觉不稳定。
3. 个人设置中的昵称输入框缺少明确语义提示。
4. 首次初始化页面仍带“主人”默认值，且缺少每月预算初始化输入。

## Decision
1. 聊天发送按钮：固定宽高并强制文案单行，避免中文换行。
2. 分段切换控件：将指示器与文本色切换改为即时更新，去除位移动画，保留外部状态记忆。
3. 个人设置：在昵称输入区增加“个人名称”标签，输入框提示改为“请输入您的名称”。
4. 首次初始化：
   - 去掉“主人”默认显示，改为占位提示“请输入AI对您的称呼”。
   - 新增“初始化每月预算”输入。
   - 初始化完成时将预算持久化到账本设置。

## 操作过程
1. 更新 `ChatScreen.kt` 输入区按钮尺寸与文本行为。
2. 更新 `ThemedSegmentedToggle.kt`，移除 `animateDpAsState/animateColorAsState` 路径。
3. 更新 `AppSettingsScreen.kt` 个人名称输入区域结构与提示。
4. 更新 `InitialSetupScreen.kt`：新增预算输入、去默认名称、回传预算。
5. 更新 `MainActivity.kt`：首次初始化完成时写入预算配置。

## 判断与取舍
### A. 为什么分段切换改为即时而不是保留过渡动画
- 收尾阶段优先稳定与可预期，避免滚动回流时“药丸块二次滑动”造成误判。
- 即时切换在账本这类工具型页面更符合信息效率。

### B. 为什么首次初始化昵称空值回退为“我”
- 需求明确拒绝“主人”作为默认展示。
- 仍需保证系统内有可用称呼，避免空字符串影响其他 UI。

### C. 为什么预算在首次初始化即写入持久层
- 预算直接影响首页/账本统计，初始化即落库可避免首次进入后再二次设置。

## Consequences
- 正向：
  - 发送按钮展示稳定。
  - 账本切换更克制、无多余动画干扰。
  - 设置页与初始化页语义更清晰。
  - 首次使用即可得到正确预算基线。
- 代价：
  - 分段切换视觉动效降低，观感更“工具化”。

## Next
1. 如后续希望恢复轻微动效，可仅保留透明度过渡，避免位移动画。
2. 可在首次初始化增加预算建议值（按收入区间）提升引导效果。
