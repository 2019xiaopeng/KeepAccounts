# ADR 027: 聊天回执金额符号、账本明细分页与药丸对齐细节修复

- 日期: 2026-03-29
- 状态: Accepted

## 背景

发布前细节回归中出现三类问题：

1. AI 聊天回执在“收入”场景下金额仍显示负号（如 `-600`），与账本页真实记录 `+600` 不一致。
2. 账本“账本明细”区块每页仍可内部滚动，形成“页中页”滚动体验。
3. 多处药丸切换组件文字视觉上轻微上偏，未严格垂直居中。

## 决策

1. 聊天记录模型新增回执交易类型字段 `receiptType`，由 `transactionId` 关联交易后回填。
2. 回执消息 `<RECEIPT>` 元数据新增 `type`（income/expense），作为渲染层兜底判断。
3. 聊天回执金额按收入/支出动态渲染正负号和颜色；“修改”入口预填类型也按回执类型传递。
4. 账本明细分页改为固定每页 5 条，移除页内 `LazyColumn` 滚动，改为静态列表 + 页码翻页。
5. 统一优化 `ThemedSegmentedToggle` 文本布局：关闭字体额外 padding，并设置最小高度，提升垂直居中一致性。

## 影响

- 聊天页与账本页的收入/支出语义完全一致。
- 账本明细交互更清晰，避免双层滚动带来的误操作。
- 药丸组件在不同文本大小下视觉对齐更稳定。

## 代价与权衡

- 聊天观察流映射增加一项类型回填，开销可接受。
- 每页固定 5 条后翻页次数增加，但换来更稳定的阅读节奏与一致交互。
