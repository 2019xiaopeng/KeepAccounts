# ADR 029: 账本明细严格固定高度与排序药丸对称布局

- 日期: 2026-03-29
- 状态: Accepted

## 背景

在前一轮修复后，账本页仍出现两个发布阻塞问题：

1. 账本明细第 5 条记录底部仍存在裁切风险。
2. 排序药丸（按时间排序/按金额排序）高亮块上下间隙不对称，顶部更贴边。

此外，分页按钮缺少明确的可点击态高亮反馈。

## 决策

1. 明细列表改为严格固定高度匹配法：
   - 单条记录强制固定高度 68dp。
   - 记录内部文字与图标使用垂直居中布局（Column(verticalArrangement = Arrangement.Center)）。
   - 列表容器使用 LazyColumn，强制高度为 68dp * 5，且禁用内部滚动。
   - 分页区独立为列表下方单独 Row，避免覆盖列表内容。
2. 排序药丸组件在账本明细区单独重写：
   - 外层容器固定高度 36dp。
   - 选中高亮块固定高度 32dp，自动形成上下对称 2dp 间隙。
   - 删除高亮块与文字定位用 padding，改为纯居中对齐。
3. 分页按钮新增动态高亮状态：
   - 可点击态：橙黄色背景（#FFA726）+ 白色文字。
   - 不可点击态：灰色背景 + 灰色文字。
   - 点击事件受 enabled 状态保护。

## 影响

- 明细区高度完全稳定，不再随数据数量变化。
- 第 5 条记录文本显示完整，底部裁切问题消失。
- 排序药丸视觉居中一致，交互状态更清晰。

## 代价与权衡

- 固定 5 槽位在少数据时保留空白，但提升了版面稳定性和可预期性。
- 排序药丸由通用组件改为明细区定制实现，维护成本略增，但可控且换来明确一致的视觉结果。
