# ADR 015: 账本分段切换稳定性、启动更新检测与图标展示策略

- Status: Accepted
- Date: 2026-03-28

## Context
收尾阶段发现四个细节问题：
1. 聊天输入区默认尺寸与发送按钮不齐，且需保持多行输入时发送按钮顶部对齐。
2. 应用图标显示被过度裁切，仅看到中心局部。
3. 应用名称需要改为中文“猫猫记账”。
4. 账本页“日历记账 / 统计报表”在滚动离屏后回到视口，药丸指示器会从左侧重启动画。
5. 需要预留启动时检测新版本（非强制更新）能力。

## Decision
1. 聊天输入区：
   - 输入框支持 1~4 行自动换行。
   - 发送按钮固定宽高并始终顶部对齐。
2. 图标展示：
   - 前景图标改为对素材做 inset 后填充，避免自适应图标安全区造成中心裁切。
3. 应用名称：
   - `app_name` 修改为“猫猫记账”。
4. 账本分段控件：
   - 增加“首次布局禁用指示器动画”策略，避免离屏回流时从 0 偏移重启动画。
5. 更新检测：
   - 启动时通过 GitHub Releases latest API 拉取 `tag_name`。
   - 与 `BuildConfig.VERSION_NAME` 比较；若有新版本弹出非强制提示框。

## 操作过程
1. 修改 `ThemedSegmentedToggle`：
   - 首次测量完成前，指示器直接定位到目标位置。
2. 新增更新检测能力：
   - `GitHubReleaseApi` + `AppUpdateRepository`。
   - `AppContainer` 注入更新仓库。
   - `MainActivity` 启动后发起检查并显示可取消弹窗。
3. UI与资源调整：
   - `ChatScreen` 输入区尺寸与对齐优化。
   - `ic_launcher_foreground.xml` 改为 inset + bitmap fill。
   - `strings.xml` 更新中文应用名。

## 判断与取舍
### A. 为什么更新检测放在启动后立即执行
- 用户能第一时间获知新版本，且实现简单。
- 保持“提示但不强制”，避免打断主流程。

### B. 为什么不直接取消自适应图标
- 现代 Android 对自适应图标支持更好，取消会影响系统一致性。
- 通过 inset 适配即可兼顾完整显示与系统规范。

### C. 为什么要单独处理分段控件首次动画
- 问题根因是离屏重组后重新测量导致指示器默认从起点动画。
- 首帧直接定位可以消除视觉跳变，不影响后续真实切换动画。

## Consequences
- 正向：
  - 聊天输入体验更稳定，发送按钮对齐一致。
  - 图标显示接近完整素材，品牌识别更好。
  - 账本分段切换视觉不再“回弹”。
  - 已具备启动更新提示能力。
- 代价：
  - 启动阶段增加一次轻量网络请求。
  - 图标仍受系统遮罩规则约束，完全无裁切不可保证。

## Next
1. 如需减少冷启动网络负担，可加入最短检查间隔（如 24h）。
2. 可增加“忽略此版本”本地标记，避免重复提示同一版本。
3. 若需更强图标一致性，可后续补充单色主题图标资源。
