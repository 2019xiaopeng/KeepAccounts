# ADR 012: 账本动效降载与聊天拟人化策略

- Status: Accepted
- Date: 2026-03-28

## Context
在已有功能可用的基础上，出现两个体验问题：
1. 账本页与主导航部分动效偏重，低端机上存在掉帧风险。
2. AI 回复多为单条长文本，语气与节奏不够贴近真人聊天。

## Decision
1. 页面级切换动效优先使用轻量淡入淡出，降低大位移滑动动画比例。
2. 降低主页面高斯模糊半径并缩短 Tab 动画时长，减少每帧渲染压力。
3. 在仓库层对 AI 普通回复进行分段（1~3 条）并分别落库，形成连续对话节奏。
4. 根据语气档位绑定温度参数（healing/tsundere/rational），增强风格差异。
5. 回执类消息保持单条，继续保留交易关联能力。

## 操作过程
1. 更新聊天仓库：
   - 增加语气温度映射。
   - 强化系统提示词。
   - 新增回复分段方法并逐条持久化。
2. 更新聊天页面：
   - 动态头部时间展示。
   - 新消息自动滚动到底部。
   - 顶部提示自动消失。
   - 发送中禁用发送按钮。
3. 更新主流程与页面参数：
   - 透传主题主色到首页/搜索/账本更多元素。
   - 降低 Tab 动画时长与背景模糊半径。
4. 更新账本页面：
   - 关键 AnimatedContent 改为轻量淡入淡出。
   - 收入相关高亮颜色更多绑定主题主色。

## 判断与取舍
### A. 为什么优先淡入淡出而不是继续滑动位移
- 滑动位移在复杂内容区切换时更易触发重布局与视觉抖动。
- 淡入淡出表达足够明确，且对低端机更友好。

### B. 为什么在 Repository 做分段而不是 UI 层拆分
- Repository 拥有完整 AI 原始回复与业务上下文，拆分后可统一持久化。
- UI 层仅负责渲染，避免展示策略与数据策略耦合。

### C. 为什么回执消息不拆分
- 回执承担结构化确认职责，拆分会降低确认效率并影响交易关联展示。

## Consequences
- 正向：
  - 动效更克制，页面稳定性和流畅度更好。
  - 聊天语气与节奏更自然，多轮体验更接近真人。
  - 主题主色绑定更一致，视觉统一性提升。
- 代价：
  - 动效表现力较激进滑动方案弱一些。
  - 回复分段规则目前基于标点和长度启发式，后续可继续优化。

## Next
1. 增加“性能模式”开关，允许用户选择更极简动效。
2. 将分段策略参数化，按消息类型和长度动态调优。
3. 为不同语气补充更清晰的文案边界与禁用词约束。
